Claude Codeとは？

Anthropic社が提供している、AIエージェントです。

  • ターミナル上で動作し、コードベースやファイル構成を自律的に理解・探索
  • 計画を立てて実行し、結果を確認しながら自己修正するエージェント型の動作
  • 高性能モデル Claude による高精度なコード生成・文章生成

自然言語で動く
AIエージェント

自然言語で指示するだけで、コードベースの理解・ファイル編集・バグ修正・テスト実行・コミットやPR作成まで行います。

アプリ開発以外の
一般業務にも使える

Skills・Agents などの機能により、資料作成や調査といった一般業務タスクの標準化・自動化にも幅広く活用できます。

Claude Code正式提供・ハンズオン実施の背景

「いま」Claude Code を学ぶべき理由が3つあります。

  • 1

    進化の真っ只中にある：AI駆動開発ツールのデファクトが Cursor から Claude Code へ移りつつあり、いま基礎を押さえておけば、今後の機能進化にそのまま乗れます。

  • 2

    社内に実例と知見がある：「玄米茶」などのプロダクトが Claude Code で開発されており、困ったときに聞ける相手と実例が社内にあります。

  • 3

    明日の業務からすぐ使える：コードを書かない業務（資料作成・調査・定型作業）でも効果が出るため、職種を問わず持ち帰れるスキルです。

目指す姿：「AIによって動く状態」

目指すところは「AIによる効率化」ではなく「AIによって動く状態」
Retrofit（後付け）の罠

既存プロセスにAIを当てはめる

  • 既存のスクラムのリズム（例：2週間スプリント）にAIを後付けする
  • 課題：AIの速度（秒単位）と人間中心のプロセス（週単位）が合わない
  • 結果：従来の開発イベントや開発フローがボトルネックとなり、AIの潜在能力を殺してしまう
Reimagine（再設計）の必要性

AIを中心に据えてプロセス全体を再設計する

  • First Principles Thinking：ソフトウェア開発の根本（意図からコードへの変換）に立ち返る
  • AI-Driven：AIを「実務」の中心として据え、プロセス全体を再設計する
AIが知的労働を担える状態になるとベロシティのような時間単位の労働生産性の指標は意味をなさなくなる。『ビジネス価値』そのものを指標とする必要がある

なぜ「AIによって動く状態」なのか？⇨「労働集約型からの脱却」

人手の量で成果が決まる「労働集約型」から抜け出すには、AIを前提に仕事の構造そのものを変える必要があります。

1

プロセスの転換

既存の進め方にAIを後付けするのではなく、生成AIを前提に開発・業務プロセスを再設計する。

2

役割の転換

「AIが実行、人間がコントロール」へ。AIが実行の主体となり、人間は監査と最終承認に軸足を移す（下表）。

3

スキルの転換

一部の詳しい人だけでなく、全社・全員がAIエージェントを使いこなす。→ その第一歩が本ハンズオン

役割AIエージェント（主体）人間（支援・監査）
意思決定戦略立案、タスク優先順位付けミッション定義、最終承認
実行コード生成、ドキュメント作成、運用物理的アクション、現場対応
管理進捗モニタリング、品質チェック倫理的監査、リスク責任
従来の『AIによる業務効率化』は部分最適に過ぎない。我々は『AIが自律的に業務を進める AI-Driven な業務プラットフォーム』を構築する。
この3つが揃うと、生成AIの進化がそのまま自社の能力向上に直結する
「AIによって動く状態」になる。
